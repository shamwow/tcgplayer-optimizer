# test

Test suite using vitest (unit + verification) and Playwright (e2e).

## Structure

- **unit/** — Pure logic tests.
- **verification/** — Tests that hit live TCGPlayer APIs. Cart API operations, listing fetches, SKU/product imports, and cart comparison workflows. These tests depend on test data that might become stale. If the tests are failing, ask if the user would like to fetch updated data. The goal of verification tests is to guide implementation against the TCGPlayer api.
- **fixtures/** — Test data: a sample cart HTML page, API listing response JSON, and a captured 72-card cart fixture (generated by `scripts/capture-live-72-card-cart.mjs`) used to avoid repeated live API calls. Might need to be updated, ask if the user would like to fetch updated data.
